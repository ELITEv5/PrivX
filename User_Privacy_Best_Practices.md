## 2. Separate Functional Wallets

Use different wallets for different purposes:

| Purpose | Recommended Practice |
|----------|----------------------|
| Trading / DEX activity | One wallet per platform or campaign |
| Savings or cold storage | Long-term address, minimal movement |
| Daily / small transfers | Fresh wallet generated regularly |
| Interacting with PrivX | Dedicated wallet only for relay activity |

This approach limits cross-linking of balances and maintains clear accounting.

## 3. Use Fresh Addresses

Most modern wallets can create multiple receiving addresses.  
Generate a **new address for each incoming transfer** when possible.  
This keeps counterparties from seeing your full balance history.

## 6. Protect Secrets Securely

Each deposit relies on a **unique secret string** that hashes into a commitment.  
Losing that secret means losing access to funds.

**Do:**
- Generate your secret offline.
- Store it in encrypted notes or password managers.
- Never reuse the same secret twice.
